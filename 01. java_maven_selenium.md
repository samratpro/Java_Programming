# Selenium Test Project Setup Guide

This README explains:

- where the original issue came from
- how to install JDK on Windows
- how to set environment variables
- which VS Code extensions to install
- how to create a Maven project from the terminal and confirm with `Y`
- how to update `pom.xml` for Selenium

## 1. Where was the issue?

The original run problem was not only one thing. It was a combination of setup issues:

1. `pom.xml` was invalid for Maven because the Selenium dependency was placed outside the `<dependencies>` section.
2. `App.java` used `WebDriverManager`, but that dependency was not added in `pom.xml`.
3. VS Code was allowed to run even when build failed because `.vscode/settings.json` had:

```json
"java.debug.settings.onBuildFailureProceed": true
```

4. The launch command used only:

```powershell
-cp target\classes
```

That classpath does not include Selenium libraries, so Java could not load Selenium classes at runtime.

5. There was also a Java version mismatch:
   - compile side used a newer JDK
   - VS Code debug runtime used Java 21

Because of that mismatch, Java can fail with `UnsupportedClassVersionError`.

## 2. What was fixed in this project?

These changes were applied:

- moved Selenium dependency inside `<dependencies>`
- set Java compile target with `maven.compiler.release`
- removed `WebDriverManager` usage from `App.java`
- changed VS Code setting so build failures stop the run

## 3. Install JDK on Windows

You need a JDK, not only a JRE.

Recommended versions for beginner Java + Selenium work:

- JDK 17
- JDK 21

### Option A: easiest setup

Use the official VS Code Java setup path:

- install the Coding Pack for Java

That bundle is recommended by the VS Code Java docs and includes VS Code, a JDK, and suggested extensions.

### Option B: install JDK manually

You can install a JDK from one of these commonly recommended sources:

- Eclipse Adoptium Temurin
- Microsoft Build of OpenJDK
- Oracle Java SE

After install, open a new terminal and check:

```powershell
java -version
javac -version
```

If both commands work, the JDK is installed.

## 4. How to set environment variable on Windows

You mainly need:

- `JAVA_HOME`
- `Path`

### GUI steps

1. Open Start.
2. Search for `Environment Variables`.
3. Open `Edit the system environment variables`.
4. Click `Environment Variables`.
5. Under `System variables`, click `New`.
6. Add:

```text
Variable name: JAVA_HOME
Variable value: C:\Program Files\Java\jdk-21
```

Use your real JDK folder path.

7. Under `System variables`, find `Path` and click `Edit`.
8. Add this entry:

```text
%JAVA_HOME%\bin
```

9. Click `OK` on all windows.
10. Close the old terminal and open a new one.

### Verify in PowerShell

```powershell
echo $env:JAVA_HOME
java -version
javac -version
```

## 5. Install Maven on Windows

Your terminal previously showed `mvn` was not recognized, so Maven is not installed or not added to `Path`.

### Install Maven

1. Download Maven binary zip from Apache Maven.
2. Extract it to a folder, for example:

```text
C:\Tools\apache-maven-3.9.14
```

3. Add this to `Path`:

```text
C:\Tools\apache-maven-3.9.14\bin
```

4. Open a new terminal and verify:

```powershell
mvn -v
```

If Maven is installed correctly, it will print Maven version, Java version, and Java home.

## 7. How to create a Maven project from terminal and press `Y`

After JDK and Maven are installed, open terminal in the folder where you want to create the project and run:

```powershell
mvn archetype:generate
```

Maven will ask questions in interactive mode.

Typical flow:

1. choose an archetype
2. enter `groupId`
3. enter `artifactId`
4. enter package name
5. review the values
6. press `Y` to confirm

### Common values

Example:

```text
groupId: selenium_test_project
artifactId: selenium_test_project
version: 1.0-SNAPSHOT
package: selenium_test_project
```

When Maven shows:

```text
Confirm properties configuration:
```

Type:

```text
Y
```

and press Enter.

### Faster command

If you want the standard Java starter project, you can use:

```powershell
mvn archetype:generate "-DarchetypeGroupId=org.apache.maven.archetypes" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DarchetypeVersion=1.5"
```

That creates a project with:

- `src/main/java/.../App.java`
- `src/test/java/.../AppTest.java`
- `pom.xml`

## 8. How to open the Maven project in VS Code

1. Open the project folder in VS Code.
2. Wait for Java extensions to load.
3. If VS Code asks to update Java/Maven configuration, accept it.
4. If something looks stuck, run:

```text
Java: Clean Java Language Server Workspace
```

## 9. How to update `pom.xml`

For this Selenium project, a clean `pom.xml` should keep all dependencies inside `<dependencies>`.

Example:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>

  <groupId>selenium_test_project</groupId>
  <artifactId>selenium_test_project</artifactId>
  <version>1.0-SNAPSHOT</version>

  <properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <maven.compiler.release>17</maven.compiler.release>
  </properties>

  <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.13.2</version>
      <scope>test</scope>
    </dependency>

    <dependency>
      <groupId>org.seleniumhq.selenium</groupId>
      <artifactId>selenium-java</artifactId>
      <version>4.20.0</version>
    </dependency>
  </dependencies>
</project>
```

### Important rules for `pom.xml`

- all libraries must be inside `<dependencies>`
- Java version should match the JDK you want to use
- if you add a library in code, add the matching dependency in `pom.xml`

## 10. Why direct `java -cp target\classes` failed

This command is not enough for Selenium:

```powershell
java -cp target\classes selenium_test_project.App
```

Why?

- it includes your compiled class
- it does not include external dependency jars like Selenium

So even if `App.class` exists, Selenium classes still cannot load.

For Maven projects, it is better to:

- run from VS Code after Maven import
- or use Maven goals from the terminal

## 11. Initial Selenium code

Use this as a beginner starter file in `App.java`:

```java
package selenium_test_project;

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class App {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();

        try {
            driver.manage().window().maximize();
            driver.get("https://www.google.com");

            System.out.println("Current URL: " + driver.getCurrentUrl());
            System.out.println("Page title: " + driver.getTitle());

            Thread.sleep(3000);
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            driver.quit();
        }
    }
}
```

### What this code does

- opens Chrome
- opens Google
- prints the current URL
- prints the page title
- waits 3 seconds
- closes the browser

## 12. How to run in VS Code

Before running, make sure:

- JDK is installed
- Maven is installed
- `pom.xml` is correct
- the project is fully loaded by the Java extensions

### Option A: right click inside `App.java`

1. Open `App.java`.
2. Right click inside the editor.
3. Look for:

- `Run Java`
- `Debug Java`

If you click `Run Java`, VS Code runs the Java main class.

If you click `Debug Java`, VS Code starts the debugger.

### Option B: use the `Run | Debug` link above `main()`

When Java extensions load correctly, VS Code usually shows CodeLens above `main()`:

```text
Run | Debug
```

Click:

- `Run` to run the program
- `Debug` to debug the program

### Option C: use the top editor button

At the top right of the editor, VS Code may show:

- a play button for running
- debug actions for debugging

### Option D: use keyboard

- press `F5` to start debugging

### Option E: use Command Palette

Open Command Palette and search for:

- `Java: Create Java Project`
- `Java: Clean Java Language Server Workspace`
- `Java: Configure Java Runtime`

## 13. If `Run Java` does not appear

Usually one of these is the reason:

1. the `Extension Pack for Java` is not installed
2. the folder is not opened as a project folder in VS Code
3. Java language server is still loading
4. the project is in lightweight mode, not standard mode
5. `pom.xml` has an error
6. JDK is missing or not configured

### Fix steps

1. Open the project folder, not only a single file.
2. Wait until Java finishes loading.
3. Install `Extension Pack for Java`.
4. Reload VS Code.
5. Run `Java: Clean Java Language Server Workspace`.
6. Accept the Maven/project import prompt.
7. Check that `java -version`, `javac -version`, and `mvn -v` all work in terminal.
